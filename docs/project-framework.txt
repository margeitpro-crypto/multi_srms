Project Framework Overview
===========================

Frontend
--------
- Library: React 19 with TypeScript
- Build tool: Vite 6 (React plugin)
- Routing: React Router DOM 7 using HashRouter
- State/context: ThemeContext, AppContext, AuthContext, DataContext, PageTitleContext
- UI patterns: Component-based layouts (PublicLayout, DashboardLayout) with route-level lazy loading (React.lazy + Suspense)
- Styling: Tailwind CSS utility classes (per components/pages) and custom components under `components/`

Backend
-------
- Runtime: Node.js + Express 5 with TypeScript (TSX for runtime transpile)
- Structure: `backend/server.ts` wiring controllers, routes, middleware, services
- Database access: `pg` pool via `config/db.ts`, environment-aware (local/test/production/supabase)
- Logging: Winston-based custom logger
- Auth/security: bcrypt for password hashing, jsonwebtoken for tokens, rate limiting middleware
- File handling & utilities: multer for uploads, nodemailer for email, xlsx/html2pdf for exports

Database & Supabase
-------------------
- Primary store: PostgreSQL (local dev) and Supabase-hosted Postgres for production
- Connection modes: direct IPv6 endpoint or Supabase session/transaction poolers (configured via `.env.supabase`)
- Migration strategy: SQL files under `supabase/migrations/` applied with `npx supabase db push`
- Seeding/testing: `scripts/test-supabase-db.ts` to verify schema, `scripts/import-seed-data.ts` for initial data

Tooling & Scripts
-----------------
- Package manager: npm (package-lock shipped)
- TypeScript config: `tsconfig.json` (ESNext module, path aliases via Vite)
- Development scripts: `npm run dev` (frontend), `npm run dev:backend` (Express with tsx watch)
- Build: `npm run build` (Vite production bundle)
- Supabase helpers: `scripts/supabase-helpers.ts` for CLI automation, numerous `npm run migrate:*` commands
- Testing/system scripts: placeholders under `scripts/` (system-test, anonymized data handling, health monitoring)

Deployment Targets
------------------
- Frontend: Vercel (static Vite build, requires `VITE_API_URL` + `GEMINI_API_KEY` envs)
- Backend/API: Any Node host (Render, Fly, Railway, etc.) with `APP_ENV=supabase` and Supabase credentials injected
- Database: Supabase project `hxfjqeoghziymujnfjpu`

Key Environment Files
---------------------
- `.env` (local dev DB + API port)
- `.env.supabase` (Supabase pooler connection string, service role, anon key)
- `.env.production`, `.env.test`, `.env.supabase.example` for other targets

This file summarizes the technical stack so new contributors can quickly understand how GitHub (code), Supabase (DB), and Vercel (UI) integrate within the project.
